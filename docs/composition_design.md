# Composition Architecture Design

## Core Insight

Composition involves a lot of **transformation of seed material**, vs. independent generation.
Central ideas of a piece (whether opening theme, hook, fugue subject, Shencker "satz", etc.) is built from one or a few seed figures that get inverted, augmented, diminuted, replicated, varied, and recombined.
Not that there aren't other passages that require creativity and artistry, but they also typically have functional value as well (pre chorus, oft-repeated ascending figures leading to a climactic cadence, passage getting us back to original key for the recapitulation of the main theme, etc.)

## Template Philosophy

A template at any level can contain any mix of:
1. **Actual material** — user-entered or previously-generated concrete content ("here's my hook")
2. **Transformation instructions** — specific ("invert figure A", "repeat figure B for every step of figure A") or general (figure A' goes here, meaning, do *something* to figure A)
3. **Constraints** — "8 bars, stepwise motion, ends on tonic, minor feel"
4. **Structural/functional directives** — "transition from Dm back to C major over 4 bars"
5. **Nothing** — composer decides freely

The specificity spectrum ranges from "reproduce Mary Had a Little Lamb exactly" to "32 bars, whatever you want." Every point in between is valid.

## Composition Phases

### Phase 1: Seeding
Generate or accept seed figures — the raw thematic material of the piece. User can:
- Enter figures directly (notation input or piano keyboard recording)
- Ask the composer to generate N seed figures with given constraints
- Mix both: enter a rhythm, let the composer find pitches (or vice versa)

### Phase 2: Planning
Decide the form, assign transformations, create the recipe. This is where the template structure gets populated:
- How many sections, what form (ABA, verse-chorus, sonata, etc.)
- Which seeds go where, what transforms apply
- Where connective/transitional passages are needed
- Cadence points, key changes, dynamic arc

### Phase 3: Realizing
Execute the recipe — transforms, connective tissue, cadential figures. The composer walks the template and fills in everything that isn't already specified or locked.

The user can intervene at any phase, lock results, and re-compose selectively.

## Data Structures

### Seeds

Seeds are the raw thematic material.
They can be MelodicFigures entered by the user or generated by the composer.
They're stored at the Piece level because multiple Parts and Sections may reference them.
In places only their Steps (melodic movement) or Pulses (rhythm) may be re-used.

```cpp
struct Seed {
    std::string name;          // "main_theme", "hook", "bridge_motif"
    MelodicFigure figure;      // the actual musical content
    bool userProvided{false};  // true = user entered, don't regenerate
    uint32_t generationSeed{0}; // for reproducibility if generated
};
```

### FigureTemplate

The atomic unit of the composition recipe. Either references a seed (with optional transformation), specifies constraints for generation, or contains a fully locked figure.

```cpp
enum class FigureSource {
    Generate,    // composer creates from constraints
    Reference,   // use a seed figure directly
    Transform,   // derive from a seed or previous figure
    Locked,      // fixed content, don't touch
};

enum class TransformOp {
    None,
    Invert,           // flip step directions
    Reverse,          // retrograde
    Augment,          // double pulse durations
    Diminute,         // halve pulse durations
    VaryRhythm,       // split/dot some pulses randomly
    VarySteps,        // randomly perturb some step values
    NewSteps,         // keep rhythm, generate new steps
    NewRhythm,        // keep steps, generate new rhythm
    Replicate,        // repeat N times with step offset between
    Combine,          // append another figure
};

struct FigureTemplate {
    FigureSource source{FigureSource::Generate};

    // For Generate: constraints
    int minNotes{2};
    int maxNotes{8};
    float totalBeats{4.0f};        // 0 = composer decides
    float defaultPulse{0.0f};      // 0 = composer decides
    bool stepwise{false};          // prefer stepwise motion
    bool skips{false};             // prefer skip-heavy motion
    int targetNet{0};              // required net pitch movement (0 = free)

    // For Reference / Transform: which seed and what operation
    std::string seedName;          // references a Seed by name
    TransformOp transform{TransformOp::None};
    int transformParam{0};         // e.g. step offset for Replicate, count for VarySteps

    // For Locked: the actual figure
    std::optional<MelodicFigure> lockedFigure;

    // Metadata
    uint32_t seed{0};              // generation seed for reproducibility
    bool locked{false};            // user has accepted this result
};
```

### PhraseTemplate

A sequence of FigureTemplates with connectors between them. Specifies how figures relate to each other within the phrase.

```cpp
struct PhraseTemplate {
    std::string name;                          // "antecedent", "consequent", "bridge"
    Pitch startingPitch;                       // where to begin (may be overridden by connector from previous phrase)
    std::vector<FigureTemplate> figures;
    std::vector<FigureConnector> connectors;   // [i] connects figure[i] to figure[i+1]

    // Phrase-level constraints
    float totalBeats{0.0f};                    // 0 = sum of figures
    int cadenceType{0};                        // 0=none, 1=half, 2=full (composer interprets)
    int cadenceTarget{-1};                     // target scale degree for cadence (-1 = composer decides)

    // State
    uint32_t seed{0};
    bool locked{false};
};
```

### PassageTemplate

What a Part plays during a Section. A sequence of PhraseTemplates. This is where large-scale thematic relationships are expressed.

```cpp
struct PassageTemplate {
    std::string name;
    std::vector<PhraseTemplate> phrases;

    // Passage-level directives
    std::string character;           // freeform hint: "energetic", "lyrical", "transitional"
    std::string fromKey;             // for modulatory passages: starting key
    std::string toKey;               // ending key (empty = same as fromKey)

    // State
    uint32_t seed{0};
    bool locked{false};
};
```

### PartTemplate

Describes one voice/instrument across all sections of the piece. Contains a PassageTemplate per Section.

```cpp
enum class PartRole {
    Melody,
    Bass,
    Harmony,
    Drums,
    Countermelody,
    Pad,
};

struct PartTemplate {
    std::string name;
    PartRole role{PartRole::Melody};
    std::string instrumentPatch;              // patch file for rendering

    // One PassageTemplate per section (keyed by section name)
    std::unordered_map<std::string, PassageTemplate> passages;
};
```

### PieceTemplate

The top-level template. Contains seeds, section structure, and part templates.

```cpp
struct PieceTemplate {
    // Musical identity
    std::string keyName{"C"};
    std::string scaleName{"Major"};
    Meter meter{Meter::M_4_4};
    float bpm{120.0f};

    // Seed material
    std::vector<Seed> seeds;

    // Structure: sections in order
    struct SectionDef {
        std::string name;            // "verse", "chorus", "bridge", "A", "B"
        float beats{32.0f};          // length in beats
        std::string scaleOverride;   // empty = use piece key
    };
    std::vector<SectionDef> sections;

    // Parts
    std::vector<PartTemplate> parts;

    // Form shorthand (optional): "AABA", "verse chorus verse chorus bridge chorus"
    // If set, composer expands this into sections automatically
    std::string form;

    // Global constraints
    uint32_t masterSeed{0};          // master seed for reproducibility
};
```

## Compose API

```cpp
struct IComposer {
    virtual ~IComposer() = default;
    virtual std::string name() const = 0;

    // Compose at different scopes — Piece always passed for context
    virtual void compose(Piece& piece, const PieceTemplate& tmpl) = 0;
    virtual void compose(Piece& piece, const PieceTemplate& tmpl,
                         const std::string& partName) = 0;
    virtual void compose(Piece& piece, const PieceTemplate& tmpl,
                         const std::string& partName,
                         const std::string& sectionName) = 0;
};
```

All three overloads take the full Piece (for context — boundary pitches, harmonic state, what's already been composed) and the full PieceTemplate (for the recipe). The narrower overloads compose only the specified scope, respecting locked content and using existing composed material as context.

### Internal Flow (for a full compose)

```
compose(piece, tmpl):
    1. Set up piece structure from tmpl (sections, parts, key, meter)
    2. Generate/accept seeds (Phase 1)
    3. For each part, for each section:
         a. Look up PassageTemplate for this part+section
         b. For each PhraseTemplate in the passage:
              - For each FigureTemplate in the phrase:
                  - If Locked: use lockedFigure
                  - If Reference: copy seed figure
                  - If Transform: apply operation to referenced figure
                  - If Generate: use FigureBuilder with constraints
              - Assemble figures into Phrase with connectors
         c. Assemble phrases into Passage
    4. Store composed Passage in Part.passages[sectionName]
```

### Selective Recompose

```
compose(piece, tmpl, "melody", "chorus"):
    1. Look up Part "melody" and Section "chorus"
    2. Find boundary conditions:
         - ending pitch of previous section's passage (if any)
         - starting pitch of next section's passage (if any)
    3. Walk the PassageTemplate for melody/chorus
    4. Skip locked phrases/figures
    5. Regenerate unlocked content (with new seeds or incremented generation seeds)
    6. Replace the composed Passage, preserving locked sub-elements
```

## JSON Example: The Specificity Spectrum

### Maximum specificity (Mary Had a Little Lamb, literally)

```json
{
    "seeds": [
        {
            "name": "mhall",
            "figure": {
                "pulses": [1, 1, 1, 1, 1, 1, 2, 1, 1, 2, 1, 1, 2],
                "steps": [0, -1, -1, 1, 1, 1, 0, -1, -1, -1, 1, 1]
            },
            "userProvided": true
        }
    ],
    "sections": [{ "name": "A", "beats": 16 }],
    "parts": [{
        "name": "melody",
        "role": "Melody",
        "passages": {
            "A": {
                "phrases": [{
                    "figures": [{
                        "source": "Locked",
                        "lockedFigure": { "pulses": [1,1,1,1,1,1,2,1,1,2,1,1,2], "steps": [0,-1,-1,1,1,1,0,-1,-1,-1,1,1] }
                    }]
                }]
            }
        }
    }]
}
```

### Same rhythm, different notes

```json
{
    "seeds": [
        {
            "name": "mhall_rhythm",
            "figure": { "pulses": [1,1,1,1,1,1,2,1,1,2,1,1,2], "steps": [] },
            "userProvided": true
        }
    ],
    "parts": [{
        "passages": {
            "A": {
                "phrases": [{
                    "figures": [{
                        "source": "Transform",
                        "seedName": "mhall_rhythm",
                        "transform": "NewSteps"
                    }]
                }]
            }
        }
    }]
}
```

### Same phrase structure, free content

```json
{
    "sections": [{ "name": "A", "beats": 16 }],
    "parts": [{
        "passages": {
            "A": {
                "phrases": [{
                    "figures": [
                        { "source": "Generate", "totalBeats": 4, "stepwise": true },
                        { "source": "Generate", "totalBeats": 4 },
                        { "source": "Generate", "totalBeats": 4, "stepwise": true },
                        { "source": "Generate", "totalBeats": 4, "cadenceType": 2 }
                    ]
                }]
            }
        }
    }]
}
```

### Just constraints

```json
{
    "keyName": "C", "scaleName": "Major", "bpm": 120,
    "sections": [
        { "name": "verse", "beats": 32 },
        { "name": "chorus", "beats": 32 }
    ],
    "form": "verse chorus verse chorus",
    "parts": [
        { "name": "melody", "role": "Melody" },
        { "name": "bass", "role": "Bass" }
    ]
}
```

The composer fills in everything — seeds, figures, phrases, passages — based on genre rules and randomness.

## Pieces Deferred
- **Genre as rules engine** — constraints and probability tables that guide the composer
- **UI** — DAW-like timeline with template/composed/rendered subtracks
- **AI composer** — model-based IComposer implementation
- **Harmonic blueprint** — chord sequences on sections, optional, composer can generate or ignore
- **Recording seeds** — capture keyboard input as seed figures
- **Part dependency ordering** — composer decides which parts to compose first
