## 3. StepGenerator v2 — Implemented but Needs Iteration

### What's done
- Gap-fill after skips (85%)
- Range regression (±10 degree boundary)
- Weighted interval distribution (55% step, 20% 3rd, 10% 4th, 8% leap)
- Post-skip reversal scaling with leap size
- Single climax enforcement (60-75% zone)
- No mid-stepwise repeated notes
- Consecutive same-direction skip blocking (with triadic exception)
- Improved targeted_sequence with gap-fill and varied intervals

### Current result
- "Better but still nothing like music"
- Too many notes, no sense of structure
- Need fundamentally different approach: named contour shapes, not smarter randomness

## 4. Composer Architecture — Deeper Changes Needed

### Shape-based figure generation (not yet implemented)
- Replace single `random_sequence` with a catalog of named shapes:
  `GetArcSequence`, `GetTriadicSequence`, `GetScalarDescentSequence`,
  `GetRepeatedNoteSequence`, `GetNeighborToneSequence`, etc.
- Composer picks shapes and combines them, like Mozart thinking
  "tonic arpeggio up, stepwise descent to half cadence"

### MelodicFunction enum (proposed, not implemented)
- Statement / Development / Transition / Cadential / Free
- Each biases rhythm AND pitch generation
- Cadential: fewer notes, longer durations, stepwise to target, repeated notes OK
- Statement: clear contour, moderate range

### Cadence rhythm (reverted)
- The 25/75 duration hack was wrong
- Right approach: cadence figures should be generated as a different rhythmic idea
  (fewer, longer notes) not post-hoc tweaked
- Tie this to MelodicFunction::Cadential

## 5. Data Model Considerations (discussed, not implemented)

### FigureUnit step=0 on first note
- Currently renderer skips unit[0].step
- DURN format treats first-token-after-| as the connector step
- Parser converts to FigureConnector for compatibility
- Future: make unit[0].step meaningful, eliminate FigureConnector for Step type
- This would simplify the model and align with DURN's design

### PulseSequence/StepSequence length mismatch
- Currently N pulses, N-1 steps (source of past bugs)
- DURN format unifies them (every token has duration + movement)
- Aligning the internal model would prevent future bugs

## 6. Instrument Rendering — Voice Stealing

### Current issue
- Back-to-back same-pitch notes (G→G) don't re-articulate cleanly
- Round-robin voice pool with additive mixing: no note-off, no muting
- Quick fix: 85% gate on NotePerformer (shortens rendered duration)
- Real fix: voice stealing — when a new note starts on the same pitch,
  mute the previous voice on that pitch. Standard MIDI/synth behavior.

## 7. Research Results (for reference)

### Melody construction rules (from research agent)
- Narmour Implication-Realization model: small intervals imply continuation,
  large intervals imply reversal
- Temperley: interval distribution is normal curve centered on 0, σ≈2-3 semitones
- Caplin formal functions: Presentation/Continuation/Cadential
- Fux: gap-fill, single climax, range ≤ 10th, 60-70% stepwise
- Empirical Markov: unison 15-20%, step 50-55%, skip 15-20%, leap 10-15%
